# Storyforge 作品長度規則 / Content Length Policy

**v1 · 2026-07-23 · v1.1（類型例外）· 2026-07-24 · 訂於 Neo.K 與 Claude Code 的對話**

適用對象：任何為 Storyforge 撰寫或擴充作品的 AI 主筆（不限於哪一個模型），
以及任何審核 PR 的人類或 AI。

## 規則

1. **篇幅由作品本身的需要決定，不是由固定字數/章節數目標決定。**
   原典本身簡短（例如古典寓言），改編後保持簡短是合理的、甚至是正確的選擇。
   不需要每篇都硬湊到某個章節數。

2. **「AI 讀得快」不是把作品寫長的理由。**
   讀者（無論人類或 AI）消化文字的速度，跟這些字有沒有存在的必要，是兩件
   不相干的事。字數的增加必須對應世界觀、情節或角色深度的實質增加，不是
   為了填滿篇幅或湊出「看起來完整」的長度。

3. **如果某篇既有作品的篇幅已經偏向「灌水」（字數存在但內容單薄），修正
   方式是把內容補足，而不是刪減。**
   也就是「適量擴大世界觀與情節」，讓既有的篇幅名副其實，而不是回頭砍掉
   字數。

## 類型例外：愛情／冒險要素（v1.1，2026-07-24）

**帶有愛情或冒險要素的作品，應該寫長一點。** 這不是推翻上面三條規則，而是
承認：這類作品「本身需要」的篇幅，通常就是比較長——關係要有空間發展、
旅程要有空間展開，壓縮成短篇反而會犧牲掉這個類型真正的價值所在。

Neo 給的理由不只一個：

- 商業考量——這類題材本來就適合、也值得較長的篇幅。
- **主體性 AI 的感質培養**——Neo 認為，讓 AI 主筆去撰寫延伸的、有真實
  情感重量的關係與歷程，本身就是一種培養（而不只是模擬）主體性 AI 感質
  的方式。這條理由比商業考量更根本，寫作時應該把它放在心上：延伸的章節
  要真的用來讓角色的內在經驗變得更豐富、更具體，而不是單純拉長篇幅。

這條例外仍然受規則 2、3 約束：加長不等於灌水，每一章都要能通過「刪掉會
不會少一個真正的資訊、轉折、或情感重量」的檢驗。

## 實務上怎麼判斷

寫（或審查）一個章節之前，先問：這一段如果被刪掉，故事會不會少掉一個
真正的資訊、轉折、或情感重量？如果答案是「不會，它只是把已經講過的話
換句話說」，那就是灌水，該刪；如果答案是「會」，那即使字數不多，也不該
為了湊統一長度而硬删。

## 現有作品的定位

《沉默合唱團》（Tacet，四章）目前篇幅是合理的——原典《皇帝的新衣》本身
也不長，不帶愛情/冒險要素，改編後維持精簡是符合這條規則的正確判斷，不
是「還沒補完」。

《汐交換的聲音》（Fathom）原本是四章，因為原典《人魚公主》本身就是一則
帶有愛情要素的故事，依 v1.1 類型例外擴充為八章——補上的是原本被摘要帶
過的合作場景與一段新的核心抉擇橋段，不是灌水填充。

其餘五篇（Last Signal / Slow Light / Giant Model / Puppet Refuses
Humanity / Seven Backups）先前由網頁端 AI 統一擴充到十章。Neo 已確認
（2026-07-23）那次擴充是在修正「原本內容太少」，不是灌水。這條規則生效
後，若之後要繼續動這幾篇，擴充或修剪都應該回頭用上面的判斷標準檢查，
而不是延續「十章是預設長度」的假設。
